◆ chunks()

chunk_t ltl::chunks ( std::size_t  n)

chunks - group elements by chunks of n elements

It can be useful to separate an array into several chunk of identical size.

std::vector<int> bigArray;
for(auto chunk : bigArray | ltl::chunks(100)) {
for(auto element : chunk) {
use(element);
}
}

Note : The last chunk may not contained n elements if there is less than n remaining elements

◆ group_by()

template<typename F >
auto ltl::group_by ( F &&  f)

group_by - group elements of an array

group_by works only on sorted array (or at least on partitionned array). Else, you could encountered some weird behaviors.

Let's say we have a list of players belonging to a team, your players are partitionned by teams and you want to iterate on teams and their players

The idea using group_by is the following one

struct Player {
std::string name;
std::string team;
};
std::vector<Player> players;
for(auto [team, players] : players | ltl::group_by(&Player::team)) {
std::cout << "Player in team " << team << " are:\n";
for(const auto &player : players) {
std::cout << " " << player.name << std::endl;
}
};

◆ map()

template<typename... Fs>
constexpr auto ltl::map ( Fs...  fs)
constexpr

map - Used to perform map operation on a functor

Apply the composition of fs to the functor. The functor can be different things.

  • It may be an array
  • It may be an optional
  • It may be an expected We use it through the operator |
std::vector<int> numbers;
// Here we get the square of the numbers as string
std::vector<std::string> strings_square = numbers | ltl::map(square, lift(std::to_string));

We may apply the operation with a monadic behaviour using operator>>

std::vector<int> f(int value);
std::vector<int> numbers;
std::vector<std::vector<int>> result = numbers | ltl::map(f);
std::vector<int> result = numbers | ltl::map(f) | ltl::join;
std::vector<int> result = numbers >> ltl::map(f);

For an array, the operator>> is equivalent to operator| and a join

◆ compose() [2/2]

template<typename F , typename... Fs>
constexpr auto ltl::compose ( F  f,
Fs...  fs 
)
constexpr

Compose the function in the left to right order.

Contrary to maths where g o f means g(f(x)), compose(g, f) will do f(g(x)) It allows to write in a more expressive and a more natural way some operations where ones would write lambda instead.

Let's say we have a Person with a name. If you want to get the size of the name of a person, you can do it in several ways. Here are three equivalents, one using compose, one without, and one trivial.

struct Person {
std::string name;
};
Person p;
auto size1 = p.name.size(); // Simple because we don't need a predicate here
auto size2 = [](const auto &person) {return person.name.size(); }(p);
auto size3 = ltl::compose(&Person::name, &std::string::size)(p);

In real-world code, you will probably never use compose directly, but indirectly within algorithms, or range features.

construct - This function returns a builder of the type T

This function may be used to convert a range of a type to another one

struct A {};
struct B {
B(A);
};
struct C {
C(int, B);
};
std::vector<A> someAs;
std::vector<B> someBs = someAs | map(construct<B>());
std::vector<C> someCs = someBs | map(construct<C>(5));

Be careful, this function is lazy
